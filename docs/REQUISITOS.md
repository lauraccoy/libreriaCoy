# Requisitos del sistema

## Requisitos funcionales

Los requisitos funcionales describen las capacidades que el sistema debe ofrecer a los usuarios.

El sistema Librería Coy debe permitir:

- Visualizar un catálogo de productos clasificados por categorías.
- Consultar la información detallada de cada producto.
- Registrar nuevos usuarios.
- Iniciar sesión de forma segura.
- Gestionar un carrito de compra añadiendo y eliminando productos.
- Realizar el proceso de checkout.
- Acceder a artículos del blog tanto desde la aplicación web como desde la app móvil.
- Consumir la información del sistema mediante una API REST.
- Autenticarse mediante Spring Security para proteger el acceso a funcionalidades privadas.

---

## Requisitos no funcionales

Los requisitos no funcionales definen los criterios de calidad que debe cumplir el sistema.

La aplicación debe garantizar:

- Seguridad en la autenticación y gestión de usuarios.
- Arquitectura escalable basada en separación de capas.
- Acceso eficiente a la base de datos mediante JPA/Hibernate.
- Diseño responsive compatible con distintos dispositivos.
- Código mantenible siguiendo buenas prácticas de desarrollo.
- Alto rendimiento en la carga de recursos estáticos.
- Baja latencia en las respuestas de la API.
- Organización modular que facilite futuras ampliaciones.

Estos requisitos han servido como base para el diseño y desarrollo del sistema, garantizando que la solución final cumpla los objetivos definidos en el plan de empresa.
